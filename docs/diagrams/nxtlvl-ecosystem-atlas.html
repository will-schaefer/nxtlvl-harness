plugins/nxtlvl-wiki

Referencenxtlvl-wiki/plugins/nxtlvl-wiki

The wiki's entire toolchain, decoupled from its content (wiki ADR-001). Three content layers: raw/ (Layer 1, immutable source notes) · wiki/ (Layer 2, synthesized pages, a citation on every claim) · CLAUDE.md (Layer 3, the schema). Ingest-family commands always finish with a lint gate then a commit — invoking the command is the commit authorization.

plugins/nxtlvl-labs

Incubationnxtlvl-labs/plugins/nxtlvl-labs

A deliberately thin, multi-provider plugin: it drives the incubation Spine from a session and never reimplements the lab — every entry point resolves harness-lab/ and shells out to bin/*.ts. Four provider manifests (root, .claude-plugin, .codex-plugin with brandColor #1F7A8C — the accent of this very page — and .devin-plugin) share one body: Claude Code gets slash commands, Codex and Devin get a byte-identical skill surface.

  • bin/ tools — new-cell, eval, graduate, pressure-report, quarantine (capture → promote appends both an eval case and a matching criterion), ledger. Node ≥24 native type-stripping, erasable syntax only, every tool tested.
  • Dogfooding symlink — harness-lab/.claude/skills → ../cells: open harness-lab as the project and every skill-cell auto-loads as a project skill. Zero duplication; cells don't leak upward.
  • vendor/ — pinned read-only snapshot of the authoring toolkit (agent-dev @ 9f27c95, skill-creator, plugin-dev, a harness-review pointer). Re-sync only; ask-first boundary.
  • labs-app — local web cockpit (Hono API :3100 + React UI :5173). Server never mutates lab files; browser actions queue in a bridge that an IDE agent drains, confirms, and executes — write-loop separation.

evals-lab — the measurement engine, and the one seam

manifest.yaml + evals/cases.yaml ──{ eval spec }──▶ evals-lab engine ──{ scorecard }──▶ graduate.ts gate

The two labs are decoupled by a data shape, not a code dependency (normative in harness-lab/docs/seam-contract.md; changing it is ask-first). Today the seam is backed by a deterministic stub; the real engine replaces it without the gate ever knowing. Scorecards carry no timestamps — byte-stable and golden-testable. A declared criterion with no eval case is passed:false, never a silent pass. Different toolchains on purpose: harness-lab TS/ESM/Node≥24, evals-lab JS/CJS/Node≥18.

The gate fails OPEN

graduate.ts exits 0 on any crash — a bug in the gate can never lock a capability out. Exit 2 is only ever a deliberate block on the three objective criteria: integrity, declared-evals-pass, intake present.

The measurement fails CLOSED

engine.evaluate() is pure and total — never throws. Any engine or adapter crash emits status:"error" + allPassed:false, so a bug can never fake a green light.

nxtlvl-wiki-mcp — the first self-authored MCP server

Read-only stdio server over the sibling wiki checkout (NXTLVL_WIKI_ROOT, default ../nxtlvl-wiki). Labs ADR-001 sets the pattern the Teams layer will reuse — read-only, stdio, cross-repo, leads-never-evidence — with the wiki plugin as its graduation target (the TS source now also sits under the wiki plugin's mcp/). Four tools, all annotated read-only and idempotent: search (ranked leads) · list (by cluster | type | tag | moc) · get_page (frontmatter, body, links, backlinks, citations) · get_source (the raw/ note — verify-at-primary). No longer registered in the root .mcp.json (that now points at remote DeepWiki); launched on demand via npm start.

Cross-cutting doctrine

Six invariants show up in every corner of the ecosystem — they are what make four plugins feel like one system.

Leads, never evidence

Every AI-generated or secondary source is firewalled to orientation: DeepWiki (harness ADR-024), the wiki MCP server, and the wiki's knowledge graph point you at a primary source you then cite. Context7 is the inverse instrument — it testifies to primary docs with a CITE stamp @ version (ADR-025).

Narrow write authority

Writers are few and named: repo-keeper is the wiki plugin's only write+commit agent; scouts propose. In production, git-workflow-runner can commit but cannot edit source; doc-keeper and evolver write only docs and evolved artifacts. The labs cockpit server never mutates — writes queue through a bridge an agent must drain.

Promotion is a git mv

Activation is a move, never a flag: sandbox/ → plugins/nxtlvl in the harness; a graduated cell → its manifest.target, carrying its evals/ with it. Stage is data (manifest.stage), not location — nothing silently goes live.

Gates are deterministic

The wiki's lint gate precedes every commit; the labs' graduation gate is three objective pass/fail criteria; taste is warnings only. And the failure modes are chosen, opposite, and intentional: the gate fails open, the measurement fails closed.

One source of truth, thin front doors

Commands are shims over shared logic: /compare and /tend-reference dispatch the same repo-keeper; /add-profile (skill) and /tend-reference profile share one job; every labs command shells out to the same bin/ tools the cockpit bridge calls.

Decisions are ADRs, kept forever

Every layer records its why in the house ADR format; superseded ADRs are kept with a superseded-by pointer, never deleted. Even drift is documented rather than hidden (D'Hondt weights superseded in code, noted in place).
