# Intent: Personal Agent Harness ("my own ecc")

> Confirmed statement of intent, produced via `interview-me` on 2026-06-16.
> This is the anchor the build is measured against. Downstream specs/plans consume it.

## One-line stack

**agent-skills = foundation + method · ecc = reference + backstop · my harness = the tailored production layer built across both.**

## Confirmed intent

- **Outcome:** My own production-quality agent harness (a Claude Code plugin) that
  reproduces ecc's full architecture and machinery — layered global/project, skills,
  agents, hooks, audit scorer, optimizer — scoped to my needs, not ecc's breadth.
- **Foundation & method:** Built **on and with agent-skills** — its conventions are the
  structural skeleton my harness extends, and its SDLC workflow
  (`/spec → /plan → /build → /test → /review`) is how I drive construction.
- **Reference & backstop:** ecc stays installed all-on as the architectural reference I
  study and the capability backstop I lean on while mine fills in.
- **User:** Me — run daily across my real work, local machine.
- **Why now:** Deliberate rebuild after wiping `~/.claude`; I want a harness I own and
  understand to the metal, not one I merely configure.
- **Success:** I run primarily on my own harness; it covers my actual workload at
  ecc-level quality; I understand every part because I reconstructed it — verified by
  audit-delta improvement from a Phase-0 baseline.
- **Constraint:** Learning-by-reconstruction — study each ecc subsystem, then build my
  own; pace set by understanding, not just shipping.
- **Out of scope:** ecc's 250-skill breadth; old SwarmLab (lifecycle / personas / gates);
  the "lean 3-rule veneer" framing — **dropped**. Catalog grows tailored to real tasks,
  not pre-built for hypothetical ones.

## Corrections captured during the interview (vs. the original rough plan)

1. **"Lean / minimal / thin layer" framing is dropped.** The goal is to *reproduce ecc*
   (full architecture + machinery at production quality), tailored to a smaller,
   need-scoped catalog — not a 3-rule personal veneer riding on top of ecc.
2. **agent-skills is the foundation AND the method** — both the structural base the
   harness extends and the SDLC workflow used to build it. (New since the rough plan.)
3. **Scope is reproduce-tailored, not match-breadth.** Full ecc-class skeleton and
   rigor now; catalog size grows organically as real work names the gaps.
4. **Machinery first, catalog reactively.** Build the (knowable, ecc-modeled) machinery
   up front; let the tailored catalog grow as actual tasks surface, with installed-ecc
   covering anything not yet built.

## Resolved unknowns

- **Plugin vs. understanding:** Both — production harness I use daily *and* a learning
  exercise. The plugin is real because I run on it.
- **"Flexible and layered":** Means the conventional two-tier global + per-project CC
  structure that coordinates — not pre-built flexibility scaffolding.
- **ecc relationship:** Permanent reference + backstop (stays installed all-on), while
  my harness becomes my primary.
