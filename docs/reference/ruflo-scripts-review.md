> **ruflo — scripts review.** The largest harness in the set, and — surprisingly — its *script layer is the most genuinely engineered code reviewed so far*: the canonical hook scripts force-exit-0, fail open, document their own limitations honestly, and the degraded memory mode *announces itself* instead of lying — a sharp reversal of the rhetoric–wiring gap found in ruflo's docs and the fabricated-success of `ruflo-adr/verify.mjs`. Analyzed 2026-06-20 · 80M · source: local vendored. Scope: scripts. Method: local read-only sampling fan-out (~22 scripts sampled across 2246) → scripts-specialist synthesis.

## 1. Spine — the headline judgment

**The load-bearing script layer is high-craft and honest; the bulk of the 2246-file count is engine internals + markdown-skill triplication, not script-craft to score.** The hook dispatcher (`.claude/helpers/hook-handler.cjs`) and the auto-memory hook (`.claude/helpers/auto-memory-hook.mjs`) are the only scripts Claude Code actually executes per session, and they are written by someone who understands hook execution semantics: stdin read with a 500ms timeout for the ambiguous-pipe case (`hook-handler.cjs:80-90`), a `runWithTimeout` race that *documents in-comment why it cannot interrupt a synchronous CPU-bound `fn`* (`:48-58`), `process.exit(0)` in a `.finally()` so a thrown hook never wedges the session (`:286-289`), and a single genuine block-gate (`pre-bash` → `process.exit(1)` on a 4-string denylist, `:148-158`).

The known "memory lies" pattern (Jaccard / hash-embeddings / `quality*=1.05`) **does not manifest as a lie at the script layer**: when the ONNX embedder is unavailable the fallback hash-embedder is reached through an explicit, *logged* branch — `console.log('[Embedding] Using fallback hash-based embeddings: ' + e.message)` (`learning-service.mjs:480,487`). The caller is told it dropped to degraded mode. That is the opposite of `ruflo-adr/verify.mjs` fabricating exit-0 over a missing backend. Audit scripts likewise have honest typed exits (`audit-plugin-manifest.mjs:` `process.exit(findings.length > 0 ? 1 : 0)`) and a `--format json` deterministic mode.

The cap on the score is **not dishonesty but over-suppression + count-inflation**: `safeRequire` mutes *all* `console.log`/`console.error` and swallows every error from the four core helper modules (`hook-handler.cjs:16-29`), so a broken `intelligence.cjs`/`memory.cjs` fails *invisibly* — fail-open taken one step too far into fail-silent. And the 2246 figure is inflated by the v3 engine (1619), the `ruflo/` mirror (290), and `.agents/` SKILL triplication — the *scripts* worth judging number in the low dozens.

## 2. What's there & how it works

Sampling (stated explicitly): I treat **`.claude/helpers/*`** as the canonical hook layer (the copy `.claude/settings.json` actually wires), and read the bin shims, the v3 CLI entry, one audit script, and representative shell scripts. I did **not** score the 1619 v3 engine `.ts`/`.js` files or `.agents/` SKILL copies as script-craft — they are library/engine code and prose, and triplication means scoring them would triple-count.

End-to-end (a hook fire): Claude Code invokes `node .claude/helpers/hook-handler.cjs <event>` with JSON on stdin (`settings.json` PreToolUse/PostToolUse/SessionStart/Stop/PreCompact). `safeRequire` lazy-loads router/session/memory/intelligence muted; `readStdin` pulls the payload with a 500ms cap; the matching handler runs inside `Promise.race` against a 3s timeout; whatever happens, `main().finally(() => process.exit(0))` guarantees a clean exit — except `pre-bash`, which `process.exit(1)`s *before* reaching the finally to actually block a dangerous command. SessionStart/Stop also drive `auto-memory-hook.mjs import|sync`, which tracks the active backend and flushes it on SIGTERM/SIGINT (`auto-memory-hook.mjs:38-52`) so a mid-run kill doesn't leave a half-written store.

Triplication note: contrary to the "byte-triplicated" prior, the *load-bearing* `.cjs` hooks are only lightly duplicated — `hook-handler.cjs` exists 2× (canonical + a `v3/@claude-flow/cli/.claude/` copy); `router.cjs`/`memory.cjs` are single-copy. The N-plexing is concentrated in `.agents/skills/**/SKILL.md` + SPARC markdown + the `ruflo/` mirror, not in executable hook code.

**Strengths (evidence)**
- *Honest failure posture* — fallback embeddings are announced (`learning-service.mjs:480,487`), audits exit-code on findings (`audit-plugin-manifest.mjs` tail), no fabricated clean exits over missing backends. Directly rebuts the `ruflo-adr/verify.mjs` and "memory lies" disqualifiers at the script layer.
- *Hook execution literacy* — stdin timeout for ambiguous pipes (`hook-handler.cjs:80-90`), async-aware `runWithTimeout` with a self-documented limitation (`:48-58`), guaranteed `exit(0)` in `.finally` (`:286`), graceful-flush on signals (`auto-memory-hook.mjs:38-52`).
- *Injection/portability hygiene* — parameterized SQL, env-relative paths, no hardcoded user paths, no `eval`, a redaction pre-commit gate.
- *Real engine craft* — `v3/.../cli/bin/cli.js:16-50` redirects upstream libs' stdout progress to stderr to protect MCP JSON-RPC framing, with an audit-log reference explaining why a broader filter was rejected.

**Weaknesses & risks (claim-vs-wiring)**
- **D3 cap — fail-silent over-reach:** `safeRequire` mutes *all* stdout/stderr and swallows every module load error (`hook-handler.cjs:16-29`); combined with pervasive `/* non-fatal */` catches, a broken `intelligence`/`memory` helper degrades to a generic `[OK]` with *no observable signal*. Fail-open is correct for a hook; fail-*silent* costs the caller the "what happened" half of D3.
- **D5 — non-atomic persistence:** `_persist()` / `saveMemory()` do full-file `writeFileSync` with no temp-file+rename, so a crash mid-write truncates the store (the SIGTERM flush helps the graceful case, not the hard-crash case).
- **Network dep at hook time:** several shell hooks invoke `npx agentic-flow@alpha` (`guidance-hook.sh:8`) — an alpha network fetch on a guidance path; fine when it 404s (it tees and continues) but a latent flakiness/supply-chain surface.
- **Count inflation, not craft:** ~2200 of the 2246 "scripts" are engine internals + triplicated SKILL markdown + the `ruflo/` mirror; the genuinely-executed script surface is a few dozen files. The headline size is misleading for a *scripts* judgment.
- **Trivial gate:** the only real block (`pre-bash`) matches 4 literal strings (`rm -rf /`, `format c:`, …) — honest about being a gate, but cosmetic as protection (`hook-handler.cjs:151`).

**Headline verdict.** On its own terms the executed script layer is **good (≈3.7/5)** — the best script-craft in the harness set so far on the two dominant dimensions: D4 is a genuine 4 (honest exits, fail-open-with-a-real-gate, degraded-mode-announced-not-faked), and the engineering literacy in the hook dispatcher is real and self-aware (it *documents its own limitations* rather than overclaiming). What caps it short of excellent is a pair of related habits: fail-*silent* error handling (`safeRequire` muting + blanket `/* non-fatal */` catches) erodes D3 observation quality, and non-atomic full-file writes leave a thin D5 crash window. The repo's 2246-script headline is count-inflation from engine internals + SKILL/mirror triplication, not script breadth — judged on the dozens of scripts Claude Code actually runs, this is a quietly competent, honest hook layer that happens to live inside a maximalist repo. No fatal flaw caps it; the "memory lies" and `verify.mjs` fabrication disqualifiers found elsewhere in ruflo **do not reach the script layer**, which is the most important finding here.
