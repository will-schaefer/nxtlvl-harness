> **CowAgent — scripts review.** A real, dogfooded Python "Agent Harness" (the renamed `chatgpt-on-wechat`) whose executable core — a streaming tool-loop with structured `ToolResult`, honest exit-code propagation, and a three-tier anti-loop guard — is genuinely strong; the only blemishes are localized (a `None`-returning swallow in the base class, one fabricated-success path) caught by defensive callers. Analyzed 2026-06-20 · 7.4M · source: local vendored. Scope: scripts. Method: local read-only sampling fan-out (~12 scripts deep-read across entrypoints / agent-loop / tools / CLI + grep sweeps over ~285 .py) → scripts-specialist synthesis.

## 1. Spine — the headline judgment

CowAgent is **not** a Claude-Code-style `.claude/` harness — it is a ~285-file **Python application** that calls itself an "Agent Harness": an IM/web chatbot framework where messages flow in through pluggable **channels** (WeChat, Slack, Telegram, Feishu, …), an **agent core** runs a plan→tool-loop over memory/knowledge/skills, and any of ~12 LLM providers generate the reply. The "scripts" domain here = the executable code that does the real work, and the real work lives in `agent/` (88 py: the tool loop, tools, memory, skills, evolution), fronted by `app.py` (process boot + channel lifecycle) and a `cow` CLI (`cli/`, click-based).

The spine judgment: **capability is in the code and the code is mostly well-engineered.** The two dimensions that dominate this rubric — observation quality (D3) and the error/exit contract (D4) — are where CowAgent is *strongest*, not weakest. The agent loop (`agent/protocol/agent_stream.py`) wraps every tool call in a structured `{status, result, execution_time}` envelope, propagates honest non-zero exit codes, forwards stderr only on failure (so the model can diagnose), truncates oversized output to a temp-file pointer, and — the standout — runs a **three-tier consecutive-failure guard** (`_check_consecutive_failures`) that distinguishes "stuck in a loop calling the same args" from "failing repeatedly" from "critical abort." That is exactly the failure *posture* this rubric rewards: it stops loud, tells the caller what happened and what to try next, and never lies a clean exit over a dead backend.

The one genuine smell that keeps it off a top score: `BaseTool.execute_tool` (`agent/tools/base_tool.py:61-65`) catches all exceptions, logs, and **returns `None`** — a latent crash, since the loop immediately dereferences `result.status` (`agent_stream.py:1195`). It is saved only because the loop's outer `try/except` (1220-1235) converts the resulting `AttributeError` into an honest error envelope. Sloppy in the base class, defended at the call site. Plus one localized dishonesty: the bash tool fabricates a success-shaped JSON error blob for a specific vision skill (`bash.py:176-185`).

## 2. What's there & how it works

| Layer | Where | What it does |
| :--- | :--- | :--- |
| Process boot | `app.py` (388 L) | Loads config, installs SIGINT/SIGTERM handlers that persist user data, syncs builtin skills → workspace, warms up MCP + scheduler on daemon threads, starts channels each in its own daemon thread via `ChannelManager`. |
| Channel lifecycle | `app.py` `ChannelManager` | Concurrent multi-channel start/stop/restart/add/remove under a lock; graceful `stop()` then 5s join then `ctypes`-based async thread interrupt as last resort (l.164-181). |
| CLI | `cli/cli.py` (81 L) + `cli/commands/*` | `cow start\|stop\|restart\|status\|logs\|update\|skill\|knowledge\|install-browser`; click groups, custom help text. |
| Shell entry | `run.sh` (55K — installer), `scripts/start.sh` (nohup), `scripts/shutdown.sh`, `scripts/run.ps1` | One-line installer + service control. `set -e`, reads from `/dev/tty` so `curl \| bash` prompts work, i18n. |
| Agent loop | `agent/protocol/agent_stream.py` (~1700 L) | `AgentStreamExecutor.run_stream`: streams model output, parses tool calls, `_execute_tool` dispatches, emits events, enforces failure guard, trims context. |
| Tool contract | `agent/tools/base_tool.py` | `BaseTool` + `ToolResult.success()/.fail()` envelope (`{status, result, ext_data}`). |
| Representative tool | `agent/tools/bash/bash.py` (451 L) | Streaming subprocess w/ progress reporting + secret redaction, tail-truncation→temp file, honest exit codes, timeout w/ process-group kill, tokenized catastrophic-command guard, Windows `$VAR`→`%VAR%` + long-`python -c` spill. |
| Chat service | `agent/chat/service.py` (374 L) | Translates agent events → CHAT socket protocol chunks; thread-safe message sync w/ careful context-trim reconciliation; cancel tokens; SQLite persistence. |

**End-to-end (web/IM turn):** `app.run()` → `ChannelManager.start()` spawns each channel thread → a channel receives a message → `ChatService.run()` builds a `Context`, attaches context-aware tools, marks the session mid-run → `AgentStreamExecutor.run_stream(query)` loops: model emits tool_call → `_execute_tool` checks `_parse_error` / consecutive-failure guard → `tool.execute_tool(args)` → structured `ToolResult` → `result_dict` re-emitted as `tool_execution_end` event → `ChatService.on_event` serializes to a `tool_calls` protocol chunk → streamed back through the originating channel; new messages persisted to SQLite, self-evolution turn noted.

**Signal vs demo:** this is production code, not demos — `run.sh` is a 55K real installer, `bash.py` handles Windows cmd.exe length limits and exit-126 retries, the loop guard exists because real LLM loops happen. Trivial/example surface is minimal (a few `__init__.py` shims, `config-template.json`).

**Strengths (evidence)**
- D3/D4 dominance done right: the anti-loop guard (`agent_stream.py:269-328`) is a textbook "fail loud, say what's next" contract with three escalating tiers and a hard critical-abort at 8 — exactly the posture this rubric prizes.
- Tool-not-found is *helpful*, not bare: it lists available tools and inlines a matching skill's SKILL.md so the model self-corrects (`agent_stream.py:1237-1266`).
- The bash tool is production-grade: streaming progress with live secret redaction, tail-truncation to a temp-file pointer, honest exit codes, process-group kill, Windows cmd.exe length-limit spill, exit-126 retry-with-argv (`bash.py` throughout).
- Robust process lifecycle: SIGTERM saves state, graceful→join→async-interrupt channel teardown, daemon-thread warmups (`app.py`).

**Weaknesses & risks (claim-vs-wiring)**
- **`base_tool.py:61-65` swallows all exceptions and returns `None`** while the caller dereferences `result.status` (`agent_stream.py:1195`) — a latent crash, masked only by the loop's outer catch. The base class should return `ToolResult.fail(...)`, not `None`. (Fatal-flaw candidate for D4, but the defended call site keeps the *observed* behavior honest, so it caps rather than sinks the score.)
- **`bash.py:176-185` fabricates a success-shaped JSON error** for the `openai-image-vision` skill — a localized lying-clean-exit baked into a general-purpose tool; it doesn't propagate widely but is the wrong instinct.
- Doc-vs-wiring is otherwise honest: README claims (planning loop, three-tier memory, MCP, channels, self-evolution) all have shipped code behind them (`agent/memory/`, `agent/tools/mcp/`, `agent/evolution/`, `channel/`). No split-brain.
- `agent_stream.py` god-module (~1700 L) and the 55K `run.sh` are audit/maintenance risks, not correctness bugs.

**Headline verdict** — On the two dimensions that decide this rubric, CowAgent's executable core is genuinely strong: the agent loop gives the caller a stable, parseable result envelope, honest exit codes, actionable next-step observations, and a disciplined three-tier failure posture that stops loud and never fabricates a clean exit over a dead backend — the bash tool in particular is production-grade. What caps it short of the top is localized and ironic: the *base class* tool wrapper swallows exceptions to `None` (a latent crash the loop happens to defend against), and one tool path fabricates a success-shaped error — two small honesty lapses in an otherwise honest, well-layered, dogfooded codebase. This is a real application's worth of competent harness-engineering, not a demo; mine the anti-loop guard and the structured-observation patterns, but note the base-class swallow as the anti-pattern to *not* copy.
