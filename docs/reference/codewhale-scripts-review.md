> **CodeWhale — scripts review.** A real, shipped (crates.io/npm v0.8.62, MIT) Rust terminal coding agent whose executable code is genuinely production-grade — typed action space, an arity-aware command-allowlist security engine, a typed multi-agent workflow IR, and CI guards that enforce docs-match-code — the rare reviewed harness where the wiring *exceeds* the docs. Analyzed 2026-06-20 · 18M · source: local vendored. Scope: scripts. Method: local read-only sampling fan-out (~22 of ~105 source units sampled — see §2) → scripts-specialist synthesis.

## 1. Spine — the headline judgment

**CodeWhale is not a `.claude/`-style harness at all — it is a from-scratch Rust coding agent (a TUI + CLI, peer to Claude Code / aider), so "scripts" here means a 15-crate Cargo workspace of Rust capability code plus a thin rind of shell/JS/Python *glue*.** Judged on general best-practice for executable agent-tool code, it is the **highest-quality executable code reviewed in the nxtlvl fan-out to date.** The spine lesson: **the security and contract discipline lives in the type system, and the docs are kept honest by executable CI guards** — the *inverse* of the "encoded N×, routed 0×, docs drift from code" pattern that dominated the ruflo/SuperClaude reviews. Where those harnesses *claimed* safety in prose and contradicted it in shipped code, CodeWhale *under-claims*: the README sells "a terminal coding agent," and the code quietly delivers a layered permission engine (`execpolicy`), workspace-escape-typed tool errors (`tools`), 0600-permission-enforcing secret storage (`secrets`), and a centralized shell-dispatch boundary that the code *comments assert* is never bypassed (`shell_dispatcher.rs:4`).

Nothing here is a fatal flaw. The only thing that caps it short of a perfect mark is that the dominant observation/error dimensions, while strong, are a coding-agent's *internal* contracts rather than the legible stdout/exit-code contracts a pure script-harness rubric idealizes — and one structural caveat (the 320K-LOC `tui` crate is a monolith) is an architecture smell, not a defect in the code I sampled.

## 2. What's there & how it works (inventory + end-to-end, cited)

**It's a Cargo workspace** (`Cargo.toml:1-19`) of 15 crates, `edition = "2024"`, `rust-version = "1.88"`, version `0.8.62`, published to crates.io and npm. Distribution: `npm install -g codewhale` downloads **SHA-256-verified** binaries (`README.md:38-40`); also cargo, Docker, Nix, scoop. 391 `.rs`, 31 `.sh`, 17 `.js`, 31 `.ts`, 8 `.py`.

**Sampling stated:** I read the load-bearing *capability* crates (`execpolicy`, `tools`, `hooks`, `secrets`, `whaleflow` + its two authoring frontends) and **all** the cross-cutting glue (the workflow authoring files, the shell smoke test, the Python CI guards, the shell-out call sites across `tui`). I did **not** read the 320K-LOC `tui` interior or every release shell script — sampled the doctrine files and call-site greps instead.

| Surface | Where | What it is |
|---|---|---|
| **Permission engine** | `crates/execpolicy/` (1432 LOC) | Layered allow/deny **prefix** ruleset (`BuiltinDefault < Agent < User`, highest-priority longest-prefix wins, `lib.rs:9-17`) + a **bash arity dictionary** (`bash_arity.rs`) so `auto_allow=["git status"]` matches `git status -s` but **not** `git push` — flags never count toward arity. Typed `ToolAskRule` with `deny_unknown_fields`. |
| **Tool action space** | `crates/tools/` (994 LOC) | Typed `ToolCapability` (ReadOnly/WritesFiles/ExecutesCode/Network/Sandboxable/RequiresApproval), `ApprovalRequirement` (Auto/Suggest/Required), and a `thiserror` `ToolError` enum whose variants include **`PathEscape { path }` — "path escapes workspace"** (`lib.rs:53`), the side-effect guard made a type. |
| **Hook/event system** | `crates/hooks/` (514 LOC) | `#[serde(tag="type", rename_all="snake_case")]` `HookEvent` enum (ResponseStart/Delta/End, ToolLifecycle, JobLifecycle, ApprovalLifecycle…) — a stable, parseable event contract for JSON logs/webhooks. |
| **Secret storage** | `crates/secrets/` (1488 LOC, `#![deny(missing_docs)]`) | `KeyringStore` trait; file/OS-keyring/in-memory backends; `config → secret store → env` precedence; **typed `InsecurePermissions { path, mode }` error when an on-disk secret isn't 0600** (`lib.rs:45-52`). No hardcoded secrets. |
| **Multi-agent workflow IR** | `crates/whaleflow/` (5716 LOC) | The "fleet/delegate" surface: a **typed workflow IR** with `branch`(parallel)/`agent`/`reduce`/`sequence`/`tournament` nodes, per-agent `mode` (read_only/read_write), `file_scope`, `isolation="worktree"`, and `budget {max_steps, timeout_secs}`. Two authoring frontends — **JS** (`workflows/issue_audit.workflow.js`) and **Starlark** (`*.star`) — **lowered to the same typed IR "without executing JS"** (`issue_audit.workflow.js:4`): authoring is data, not code execution. |
| **Shell-out boundary** | `crates/tui/src/shell_dispatcher.rs` | Single documented entry point; module doc asserts the TUI "**never** calls `Command::new("cmd")` … directly" (`:4-6`), handling per-shell quoting and crossterm raw-mode save/restore. |
| **Glue: smoke test** | `scripts/mobile-smoke.sh` | `set -euo pipefail`, `trap cleanup EXIT`, free-port picker, quoted vars, real HTTP assertions against the built binary. |
| **Glue: CI drift guards** | `scripts/check-provider-registry.py`, `check-coauthor-trailers.py` | Parse the Rust source and assert `docs/PROVIDERS.md` still matches shipped `ProviderKind`/registry IDs — **docs-match-code enforced executably.** |

**End-to-end:** agent proposes a tool call → `tools` types it (capability + approval req) → `execpolicy` checks layered prefix rules with arity-aware matching → on shell, `shell_dispatcher` builds the `Command` with correct quoting → `hooks` emits typed lifecycle events for logging/UI. WhaleFlow sits above: a declarative workflow (JS/Starlark) lowers to typed IR that fans agents out in parallel with scoped read/write modes and worktree isolation.

## 3. Specialist scorecard

| Dim | Score | Justification (file:line) |
|---|---|---|
| D1 Input/interface contract | **5** | Inputs are Rust types, not grab-bags: `ToolCapability`/`ApprovalRequirement` enums (`tools/lib.rs:18-43`), `ToolAskRule` with `#[serde(deny_unknown_fields)]` (`execpolicy/lib.rs:78`), typed WhaleFlow node IR (`whaleflow`). |
| D2 Deterministic output shape | **5** | `HookEvent` is a tagged `snake_case` enum — stable, parseable across all paths (`hooks/lib.rs:18-20`); workflow authoring lowers to one typed IR regardless of JS-vs-Starlark frontend. |
| D3 ⭐ Observation quality | **4** | Typed `ToolError` variants *name the failure and constrain next step* — `PathEscape`, `Timeout{seconds}`, `PermissionDenied` (`tools/lib.rs:46-63`); hook events carry phase + payload. Short of 5 only because these are internal-contract observations, not the legible operator-facing stdout a script rubric idealizes. |
| D4 ⭐ Error & exit-code contract | **5** | `thiserror`/`Result` throughout; **no lying clean exits** — `secrets` raises typed `InsecurePermissions` rather than silently trusting a world-readable key (`secrets/lib.rs:45`); CI guards `sys.exit` on drift; smoke test counts PASS/FAIL under `set -e`. Failure posture is fail-loud and honest. |
| D5 Side-effect safety & idempotence | **5** | Workspace-escape is a *typed error* (`PathEscape`); tools tagged ReadOnly vs WritesFiles vs ExecutesCode (`tools/lib.rs:20-30`); WhaleFlow `mode=read_only` + `isolation=worktree` quarantine writes; smoke test `trap cleanup EXIT` kills the spawned server. |
| D6 Portability & hygiene | **5** | All deps declared in workspace `Cargo.toml`; **no hardcoded `/Users`//`/home` paths or secrets in any script** (grep clean); shell-out passes args via `.arg()` not string-interpolation (`tui/hooks.rs:648-649`), and the *only* `sh -c` sites are the deliberate, allowlist-gated command-execution boundary of a coding agent — not naive injection. SHA-256-verified binary downloads (`README.md:38`). |
| D7 Cohesion & composition | **4** | Crate split is exemplary one-job-per-module (execpolicy ≠ tools ≠ secrets ≠ hooks ≠ whaleflow); authoring composed via shared IR. Capped at 4 by the **320K-LOC `tui` monolith** (`config.rs` alone 12K, `ui.rs` 11K) — a single crate carrying ~97% of LOC is a cohesion smell at the workspace level even if the small crates are clean. |

**Strengths (evidence)**
- **Security-as-types, not security-as-prose.** `execpolicy`'s arity dictionary (`bash_arity.rs:1-26`) solves the exact false-positive class nxtlvl's own dangerous-bash gate hit (`git branch -f` over-blocking) — flags excluded from prefix matching by construction, with the invariant documented and (per file layout) test-backed.
- **The docs are kept honest by executable guards** (`check-provider-registry.py`, `check-coauthor-trailers.py`) — claims-vs-wiring drift is a *CI failure*, the structural inverse of every prior harness reviewed.
- **Injection-safe workflow authoring**: JS workflows are "lowered to typed IR **without executing JS**" (`issue_audit.workflow.js:4`) — declarative config, not eval-of-untrusted.
- **Glue scripts meet shell best-practice**: `set -euo pipefail` + `trap` cleanup + quoted expansions + free-port allocation (`mobile-smoke.sh:11-37`).

**Weaknesses & risks (claim-vs-wiring; no fatal flaw)**
- **No claim-vs-wiring *gaps* found** — unusually, the code over-delivers vs the README. The honest caveat is the reverse: the `shell_dispatcher` "never bypassed" guarantee is a *doc-comment assertion* (`:4-6`), enforced by convention/review, not a compiler-level singleton — a determined contributor could still `Command::new("sh")` elsewhere (and `tui/hooks.rs:647`, `mcp.rs:4906` do legitimately spawn `sh` for their own boundaries).
- **`tui` monolith** (320K LOC, 97% of source) is an architecture risk for maintainability/testability, though not a defect in the executable contracts sampled.
- **Naming residue**: keychain service still `"deepseek"` and `DEEPSEEK_*` legacy env aliases (`secrets/lib.rs:22-29`) — deliberate back-compat, documented, but a small honest-naming wart post-rename.

**Headline verdict.** On the 7-dimension scripts rubric this is a **4.7/5 — the strongest executable code in the fan-out.** The dominant dimensions D3/D4 are excellent (typed, fail-loud, no lying exits), and D1/D2/D5/D6 are textbook: security and side-effect safety are encoded in the type system (`PathEscape`, `InsecurePermissions`, arity-aware allowlists), distribution is SHA-verified, glue scripts follow shell best-practice, and — uniquely — CI guards make docs-drift a build failure. What keeps it off a perfect score is not a flaw but a framing mismatch (its world-class contracts are a coding-agent's *internal* type-level contracts, scoring 4 on a rubric that prizes operator-facing stdout) plus one genuine cohesion smell — the 320K-LOC `tui` crate holding nearly all the code. For nxtlvl this is the reference repo to **mine for doctrine, not collide with**: the `execpolicy` arity-dictionary is a directly relevant upgrade path for the dangerous-bash gate, and "enforce docs-match-code with an executable CI guard" is a portable hygiene lesson.
