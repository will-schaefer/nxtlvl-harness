# Approved acronyms

**Scope:** the exhaustive allow-list for `plain-language.md`'s "universal industry acronym"
exception. If a term isn't on this list, spell it out — see that rule for the full policy.

Every entry still gets **spelled out on first use per document** (e.g. "ADR (architecture
decision record)"); this list only says which terms are allowed to exist as acronyms at all.

| PR | pull request |

| CD | continuous deployment |

## Not on this list — write it out

Anything coined for this project or a specific tool, even if it feels "well known" inside a
session: no "SoT" (source of truth), no invented feature nicknames, no compressed internal
jargon. When in doubt, it isn't on this list, so spell it out.

## Maintaining this list

Add an entry only when a term is genuinely universal — a working developer outside this
project would read it without pausing. Proposing an addition is itself a small decision;
just edit this file when it comes up, no ADR needed (this is a reference list, not an
architectural decision).
