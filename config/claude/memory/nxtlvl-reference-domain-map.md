The **capability companion** to [[nxtlvl-reference-repo-map]] (structural) at
docs/reference/reference-domain-map.md. A matrix of **13 nxtlvl domains × 13 harnesses** (rows =
nxtlvl's own Layer-2 workflows + Layer-3 subsystems, deliberately NOT a neutral catalog) + per-repo
domain-profile capsules + a **mining shortlist per nxtlvl surface** (who to mine ★ vs learn-against).

**Two caveats baked in:** (1) counts ≠ quality — cells seeded from a name keyword-scan then corrected
by prior deep reviews (¹ = high surface/low wiring: ruflo ideation no-router + memory lies, SC modes
dead); (2) code-runtimes under-counted by name-scan (² = capability in Python not *.md: deepagents
planning+filesystem-memory+subagent-spawn, hive DAG+role-memory, CowAgent plans+long-term-memory).

**Headline: Trellis is the un-reviewed standout** — only reference repo whose SPINE (spec+plan+memory
persisted into the repo) mirrors nxtlvl's, and it independently covers ideation (real `brainstorm` +
`first-principles-thinking` skills), hooks (59 files), and skill-authoring. **If one deep harness-review
runs next, it's Trellis.** Two corrections to the structural map: Trellis has real ideation (not just
spec); CodeWhale is NOT a pure skip — ships skill/plugin/mcp-authoring suite + gh-ops + fleet/delegate.
Mining shortlist: ideation→Trellis/ECC; memory→Trellis/hive/deepagents (ruflo=anti-pattern); orchestration→
deepagents+hive (code contrast); agent-building→CodeWhale/Trellis/agents-main. See [[nxtlvl-harness-adopt-backlog]].
