nxtlvl's intent was re-derived via `interview-me` on 2026-06-28 (see
`docs/intent/personal-harness.md`), reshaping the build before it goes further.

**The real why-now (non-obvious, drives everything):** the user lost their job (≈2026-06-21)
and is strongly considering **starting their own AI agent company.** nxtlvl is the
**production-grade foundation they'd build that company on** — initially a personal
force-multiplier, with potential to become a product itself (keep clean boundaries, no
personal-only lock-in). Learning-by-reconstruction is demoted from *goal* to *method*; the
bar is now "bet-a-company-on" production quality, not "good enough for me."

**Three original mechanisms were dropped/replaced** (they were welded to the now-removed
ecc-dormant model and died in contact with the build):
- fallback-rate north-star metric → **deliberate coverage assessment vs `nxtlvl-wiki`** (no auto metric)
- ecc-as-dormant-backstop → **native CC backstop + `nxtlvl-labs` incubation**; ecc = ingested corpus only
- fallback-fed one-line intake gate → **`nxtlvl-labs` buffer + `spec→plan→ADR`** (an audit found only 1/12 shipped components had the old intake entry; spec→plan→ADR was the de-facto gate that actually held)

**New build posture (inverts the old "whitelist, not full"):** map a pretty-complete agent-harness
architecture up front as the standing coverage map, then fill it incrementally + reactively. Next
steps: a `grill-me` hardening pass on the intent, then produce the architecture outline/spec.
