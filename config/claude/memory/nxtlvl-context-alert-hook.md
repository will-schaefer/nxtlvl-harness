A doubt-driven adversarial review (2026-06-17) corrected the spec's "just lighten it" framing: the code delta is a **small rebuild, not a tune** —
- `buildMessage` must be **replaced wholesale**, not softened: the live text is a full wind-down/checkpoint/stop directive — the exact interruption the new design retires.
- The state machine must go from `{alerted: bool}` to **two-stage**: the single boolean can't arm/fire the 200K ping and the ~325K backstop independently ("add arm-point" is not additive).
- The default-threshold fix touches **4 sites**, not 2: constant `180000` AND `hooks.json` "default 180k" disagree with the header/JSDoc `200000`; fixing only the constant leaves `hooks.json` wrong.
- New validate-before-trust items: **PostToolUse `additionalContext` actually surfaces to the model on the shipping path** (spec only flagged it for the Stop alt — if dropped, the whole signal is a silent no-op); and **native ~900K is an assumption, not a fact** (may be a fraction of window). `osascript` fail-open must be **designed** (fire-and-forget, swallow errors, don't eat the 10s timeout), not just asserted.

Key shifts from the original refine:
- **Window reality:** user runs a 1M model ([[user-1m-context-degradation]]); native auto-compaction (~900K) is a distant backstop, ~700K past the quality cliff — so the hook is NON-redundant and is the only thing acting in the quality zone.
- **Awareness-only, not behavior-forcing:** the agent must NOT stop, checkpoint, commit, or wind down. It finishes the current step and appends ONE plain line at its next report (`Context ~200K — good to /compact at the next break`), plus a macOS `osascript` notification. See [[harness-hooks-inform-not-force]].
- **Threshold lifted 180K → 200K** (also fixes the latent header-comment/constant mismatch). Single ping + one blunter escalation backstop at ~325K, then silent. Re-arm hysteresis retained.
- **Hook 2 (PreCompact) shrunk to POINTER-FIRST:** preserve a pointer to the active `docs/plan/` doc + next task through the summary (brief next-step only when no plan doc). NO agent-written handoff file — `docs/plan/` is the durable artifact. Retired: wind-down, auto-commit ("commit only if step done" → now normal task-boundary workflow, not hook-triggered), graded soft-line, handoff file.

**Event choice RESOLVED (D-event):** stay on PostToolUse (current/shipping path, `additionalContext` documented-supported); Stop hook noted as a future cost-optimization, not pursued now. **Load-bearing assumption (still open):** the "don't stop, just inform" stance bets that future workflow rules scope tasks under budget; that rule doesn't exist yet — tracked as a deferred validation item in the plan.

**Why:** quality-first, compose-on-native, minimal coupling ([[compose-on-native-quality-first]]).

**How to apply:** not part of the M0–M7 plan ([[nxtlvl-m0-status]]); it's a context-policy component (cf. ADR-007). Don't re-derive platform facts — see [[cc-context-hook-facts]]. Anchor: [[nxtlvl-harness]].
