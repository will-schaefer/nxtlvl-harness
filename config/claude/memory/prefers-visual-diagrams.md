The user stated plainly: "yes seeing it will always help me with everything." They want
architecture maps, data-flow diagrams, comparisons, and other visuals rendered (inline via
the visualize show_widget tool, or the brainstorming visual companion) rather than described
in prose alone.

**Why:** They process structure better when shown than told; this generalizes across all work,
not just one brainstorm.

**How to apply:** Default to producing a diagram/visual for anything with structure —
architectures, flows, state machines, option comparisons, timelines. Don't wait to be asked;
don't gate on token cost (they're on Max — see [[user-max-subscription]]). Still pair the
visual with a tight prose anchor. Related: [[nxtlvl-harness]].
